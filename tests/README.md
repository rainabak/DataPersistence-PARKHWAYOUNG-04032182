# 테스트 전략 — DataPersistence PoC

이 프로젝트는 PoC 단계로, 자동화 테스트 프레임워크 없이 수동 검증 시나리오로 진행한다.

---

## 테스트 목표

| 번호 | 검증 항목 |
|---|---|
| 1 | CRUD 각 연산이 정상 동작하는가 |
| 2 | 재실행 후 데이터가 유지되는가 |
| 3 | `nextId`가 재실행 후에도 연번을 유지하는가 |
| 4 | 파일이 없을 때 정상적으로 초기화되는가 |
| 5 | 파일이 손상되었을 때 앱이 안전하게 시작되는가 |
| 6 | 쓰기 중 비정상 종료 시 파일이 손상되지 않는가 |

---

## 수동 검증 시나리오

### 시나리오 1 — 기본 CRUD 검증

```
1. 앱 실행
2. Item 등록: name=Alpha, price=100
3. Item 등록: name=Beta, price=200
4. 목록 조회 → Alpha, Beta가 보여야 한다
5. Alpha 수정: price=150
6. 목록 조회 → Alpha의 가격이 150이어야 한다
7. Beta 삭제
8. 목록 조회 → Alpha만 남아야 한다
```

성공 기준: 각 단계에서 UI에 정확한 데이터가 표시된다.

---

### 시나리오 2 — 재실행 후 데이터 유지 검증

```
1. 앱 실행
2. Item 등록: name=Gamma, price=300
3. 앱 종료
4. 앱 재실행
5. 목록 조회 → Gamma가 그대로 보여야 한다
```

성공 기준: 재실행 후에도 등록한 데이터가 남아 있다.

---

### 시나리오 3 — nextId 연번 유지 검증

```
1. 앱 실행
2. Item 등록 → id=1 부여됨
3. Item 등록 → id=2 부여됨
4. id=2 삭제
5. 앱 종료
6. 앱 재실행
7. Item 등록 → id=3이어야 한다 (id=2 재사용 금지)
```

성공 기준: 재실행 후 ID가 1부터 재시작되지 않는다.

---

### 시나리오 4 — 파일 없음 상태에서 정상 초기화 검증

```
1. data/items.json 파일을 수동으로 삭제
2. 앱 실행
3. 에러 없이 시작되어야 한다
4. 목록 조회 → 빈 목록이 보여야 한다
5. Item 등록 → 정상적으로 동작해야 한다
```

성공 기준: 파일 없음을 에러로 처리하지 않고 빈 초기 상태로 시작한다.

---

### 시나리오 5 — 파일 손상 처리 검증

```
1. data/items.json을 텍스트 편집기로 열어 내용을 깨뜨린다
   예: 중괄호 삭제, 임의 문자 삽입 등
2. 앱 실행
3. 경고 메시지가 출력되어야 한다
4. 에러 없이 빈 데이터로 시작되어야 한다
5. data/items.json.bak 파일이 생성되어야 한다
```

성공 기준: 손상 파일이 있어도 앱이 중단되지 않으며, 원본이 보관된다.

---

### 시나리오 6 — 임시 파일 잔류 처리 검증

```
1. data/items.json.tmp 파일을 수동으로 생성
2. 앱 실행
3. 에러 없이 시작되어야 한다
4. items.json.tmp 파일이 자동으로 삭제되어야 한다
```

성공 기준: 잔류 임시 파일이 있어도 앱이 정상적으로 시작된다.

---

## 파일 상태 검증 방법

각 시나리오 후 `data/items.json` 파일을 텍스트 편집기로 열어 내용을 직접 확인한다.

확인 항목:
- `nextId`가 올바른 값인가
- `items` 배열에 정확한 데이터가 들어 있는가
- JSON 형식이 손상되지 않았는가

---

## 테스트 환경

- 운영체제: Windows 10/11
- 빌드: x64 Debug
- 실행 파일 위치: `x64/Debug/DataPersistence.exe`
- 데이터 파일 위치: `x64/Debug/data/items.json`

---

## 향후 자동화 테스트 전환 시 고려사항

PoC 검증 완료 후 자동화 테스트를 추가할 때는 다음을 고려한다.

- `IItemRepository` 인터페이스를 활용한 Mock 구현체로 Controller 단위 테스트
- `JsonFileStore`의 직렬화/역직렬화 로직 단위 테스트
- 임시 파일/손상 파일 시나리오를 파일 시스템 격리 환경에서 자동화
