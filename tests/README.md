# 테스트 가이드 — ConsoleMVC PoC 1

PoC 1은 별도의 테스트 프레임워크 없이 **콘솔 수동 테스트**로 검증한다.

---

## 테스트 실행 방법

1. Visual Studio에서 `Ctrl+F5` 로 프로그램 실행
2. 콘솔 메뉴가 출력되면 아래 시나리오 순서대로 입력
3. 각 시나리오의 기대 결과와 실제 출력을 비교

---

## 테스트 시나리오

### TC-01 아이템 추가 (정상)

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 메뉴 선택 | `1` | 이름 입력 프롬프트 출력 |
| 이름 입력 | `사과` | 가격 입력 프롬프트 출력 |
| 가격 입력 | `1000` | "추가되었습니다" 메시지 출력 |
| 메뉴 복귀 | — | 메인 메뉴 재출력 |

---

### TC-02 아이템 목록 조회 — 데이터 있음

**전제:** TC-01 이후

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 메뉴 선택 | `2` | `[1] 사과 | 1000` 출력 |

---

### TC-03 아이템 목록 조회 — 데이터 없음

**전제:** 프로그램 최초 실행 직후

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 메뉴 선택 | `2` | "아이템이 없습니다" 안내 메시지 출력 |

---

### TC-04 아이템 수정 (정상)

**전제:** TC-01로 ID=1 아이템 존재

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 메뉴 선택 | `3` | ID 입력 프롬프트 출력 |
| ID 입력 | `1` | 이름 입력 프롬프트 출력 |
| 이름 입력 | `바나나` | 가격 입력 프롬프트 출력 |
| 가격 입력 | `2000` | "수정되었습니다" 메시지 출력 |
| 목록 확인 | `2` | `[1] 바나나 | 2000` 출력 |

---

### TC-05 아이템 수정 — 존재하지 않는 ID

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 메뉴 선택 | `3` | ID 입력 프롬프트 출력 |
| ID 입력 | `999` | "해당 아이템을 찾을 수 없습니다" 오류 메시지 출력 |

---

### TC-06 아이템 삭제 (정상)

**전제:** TC-01로 ID=1 아이템 존재

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 메뉴 선택 | `4` | ID 입력 프롬프트 출력 |
| ID 입력 | `1` | "삭제되었습니다" 메시지 출력 |
| 목록 확인 | `2` | "아이템이 없습니다" 출력 |

---

### TC-07 아이템 삭제 — 존재하지 않는 ID

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 메뉴 선택 | `4` | ID 입력 프롬프트 출력 |
| ID 입력 | `999` | "해당 아이템을 찾을 수 없습니다" 오류 메시지 출력 |

---

### TC-08 복수 아이템 추가 후 목록 확인

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 아이템 추가 1 | `1` → `사과` → `1000` | 추가 성공 |
| 아이템 추가 2 | `1` → `바나나` → `2000` | 추가 성공 |
| 아이템 추가 3 | `1` → `포도` → `3000` | 추가 성공 |
| 목록 조회 | `2` | ID 1, 2, 3 세 항목 모두 출력 |

---

### TC-09 프로그램 종료

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 메뉴 선택 | `0` | 프로그램 정상 종료 |

---

### TC-10 잘못된 메뉴 입력

| 단계 | 입력 | 기대 결과 |
|---|---|---|
| 메뉴 선택 | `9` | "잘못된 입력입니다" 메시지 출력 후 메뉴 재출력 |
