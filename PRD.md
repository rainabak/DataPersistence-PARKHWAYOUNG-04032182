# PRD — ConsoleMVC PoC 1

Product Requirements Document  
버전: 1.0 | 작성일: 2026-05-08

---

## 1. 목적

MVC(Model-View-Controller) 패턴의 레이어 역할 경계를 C++ 콘솔 애플리케이션으로 시연한다.  
이후 PoC 2(파일 저장), PoC 3(DB 연동) 확장을 위한 인터페이스 기반 구조를 확보한다.

---

## 2. 범위

| 구분 | 내용 |
|---|---|
| In Scope | Item 엔티티 CRUD, MVC 레이어 분리, 인메모리 저장 |
| Out of Scope | 파일 저장, DB 연동, JSON 직렬화, 외부 라이브러리, GUI |

---

## 3. 사용자 시나리오

단일 사용자가 콘솔 메뉴를 통해 Item을 관리한다.

```
[메인 메뉴]
1. 아이템 추가
2. 아이템 목록 조회
3. 아이템 수정
4. 아이템 삭제
0. 종료
```

---

## 4. 기능 요구사항

### FR-01 아이템 추가 (Create)
- 사용자가 이름과 가격을 입력한다
- 시스템이 자동으로 고유 ID를 부여한다
- 추가 성공 메시지를 출력한다

### FR-02 아이템 목록 조회 (Read)
- 저장된 모든 아이템을 목록으로 출력한다
- 아이템이 없는 경우 안내 메시지를 출력한다
- 출력 형식: `[ID] 이름 | 가격`

### FR-03 아이템 수정 (Update)
- 사용자가 수정할 아이템의 ID를 입력한다
- 존재하지 않는 ID 입력 시 오류 메시지를 출력한다
- 새 이름과 가격을 입력받아 해당 아이템을 갱신한다

### FR-04 아이템 삭제 (Delete)
- 사용자가 삭제할 아이템의 ID를 입력한다
- 존재하지 않는 ID 입력 시 오류 메시지를 출력한다
- 삭제 성공 메시지를 출력한다

### FR-05 종료
- 메뉴에서 0을 선택하면 프로그램을 종료한다

---

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 언어 표준 | C++17 이상 |
| 빌드 환경 | Visual Studio 2022, Console Application |
| 외부 의존성 | 없음 (STL만 사용) |
| 데이터 지속성 | 런타임 인메모리 한정 (프로그램 종료 시 데이터 소멸) |
| 레이어 격리 | Model은 View/Controller를 포함하지 않음 |

---

## 6. 엔티티 정의

### Item

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | `int` | 자동 증가 고유 식별자 |
| `name` | `std::string` | 아이템 이름 |
| `price` | `int` | 아이템 가격 |

---

## 7. 클래스 책임 요약

| 클래스 | 레이어 | 책임 |
|---|---|---|
| `Item` | Model | 데이터 구조체 |
| `ItemRepository` | Model | CRUD 연산 (인메모리) |
| `IView` | View | 입출력 추상 인터페이스 |
| `ItemView` | View | 콘솔 입출력 구현 |
| `IController` | Controller | `run()` 진입점 인터페이스 |
| `ItemController` | Controller | CRUD 흐름 조율 |

---

## 8. 완료 기준

- [ ] Visual Studio 빌드 오류 0건
- [ ] 메뉴 → 추가 → 목록 조회 → 수정 → 삭제 → 종료 흐름 수동 확인 통과
- [ ] Controller에 `std::cout` / `std::cin` 직접 호출 없음
- [ ] Model이 View / Controller 헤더를 `#include`하지 않음
